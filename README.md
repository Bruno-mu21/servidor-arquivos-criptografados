O trabalho consistiu no desenvolvimento de um sistema cliente-servidor chamado Servidor de Arquivos Criptografados, cujo objetivo foi permitir o envio, o recebimento e o armazenamento seguro de arquivos por meio de técnicas de criptografia, troca de chaves e autenticação. Para garantir a confidencialidade dos dados transmitidos, a comunicação entre cliente e servidor foi implementada utilizando sockets combinados com criptografia simétrica, sendo obrigatória a utilização de três tipos diferentes de algoritmos simétricos, como AES, DES ou 3DES. Como a criptografia simétrica depende de uma chave secreta compartilhada, o projeto exigiu a implementação de duas formas distintas de troca de chaves segura: uma baseada em modelos PKI ou criptografia de curva elíptica (ECC), utilizando chaves públicas e privadas para cifrar a chave simétrica; e outra utilizando o protocolo Diffie-Hellman, que permite gerar uma chave em comum sem que ela seja transmitida diretamente pela rede. Além da proteção da comunicação, o aplicativo deveria oferecer um conjunto mínimo de funcionalidades, incluindo enviar arquivo, baixar arquivo, listar arquivos disponíveis no servidor e realizar a identificação do emissor, garantindo que o sistema reconhecesse qual cliente estava estabelecendo a conexão. Para completar a camada de segurança, o trabalho também incluiu um sistema de autenticação de usuário, no qual as senhas não podiam ser armazenadas em texto puro; em vez disso, deveriam ser guardadas utilizando funções de hash criptográfico como MD5 ou SHA, permitindo que o servidor validasse o login comparando hashes de forma segura. Assim, o projeto reuniu e aplicou diversos conceitos essenciais de Segurança da Informação — criptografia simétrica e assimétrica, troca de chaves, autenticação e hashing — resultando em um sistema simples, mas completo, para transmissão segura de arquivos.
